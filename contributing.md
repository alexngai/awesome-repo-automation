# Contributing

Thank you for your interest in contributing to this awesome list! Here are some guidelines.

## Adding an Item

- Search previous suggestions before making a new one to avoid duplicates.
- Make an individual pull request for each suggestion.
- Use the following format: `- [Name](URL) - Description ending with a period.`
- New categories or improvements to existing categorization are welcome.
- Keep descriptions short, simple, and objective.
- Start the description with a capital letter and end with a period.
- Check your spelling and grammar.
- The pull request title should follow the format: `Add Name of Tool`.

## Quality Standards

To be included, a project should meet these criteria:

- It must be related to git repository automation.
- It should be actively maintained (not archived or abandoned).
- It should have documentation or a README explaining what it does.
- If open source, it should have a reasonable number of stars or community adoption.
- Commercial tools should have a free tier or open-source component.
